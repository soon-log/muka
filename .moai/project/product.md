# MUKA (뮤카) 프로젝트 개요

## 프로젝트 비전

MUKA(뮤카)는 **친구가 직접 골라주는 음악 추천 서비스**입니다. 알고리즘이 아닌 사람 사이의 진정한 음악적 연결을 추구하며, 친구에게 음악 추천을 요청하고 아름답게 디자인된 음악 카드로 답변을 받는 경험을 제공합니다.

### 핵심 가치

| 가치          | 설명                                         |
| ------------- | -------------------------------------------- |
| 알고리즘 프리 | 추천 알고리즘이 아닌 친구가 직접 선택한 음악 |
| 진정성        | 타인의 시선으로 새로운 음악 발견             |
| 간편함        | 로그인 없이 링크만으로 음악 공유             |
| 감성적        | 따뜻하고 플레이풀한 카드 디자인              |

---

## 대상 사용자

### 주요 타겟

- **연령대**: 20-30대 MZ 세대
- **특성**: 음악을 좋아하고, 친구와의 소통을 중요시하는 사람들
- **사용 환경**: 모바일 중심 (카카오톡 사용자)

### 사용자 페르소나

**A (발신자)**: 친구에게 음악 추천을 받고 싶은 사람

- 새로운 음악을 발견하고 싶음
- 친구가 자신을 어떻게 생각하는지 호기심
- 재미있는 콘텐츠로 친구와 소통하고 싶음

**B (수신자)**: 음악 추천 요청을 받은 사람

- 좋아하는 음악을 친구에게 공유하고 싶음
- 간편하게 추천할 수 있는 방법을 선호
- 예쁜 카드로 센스 있게 추천하고 싶음

## 사용 사례 및 시나리오

### 시나리오 1: 일상적 호기심

> "친구가 나를 생각하면 어떤 음악이 떠오를까?"

1. A가 질문 템플릿 "나를 생각하면 떠오르는 음악은?" 선택
2. 링크 생성 후 카카오톡으로 B에게 전송
3. B가 링크를 열고 음악 검색
4. B가 카드 디자인 선택 후 공유
5. A가 카드 확인 및 음악 감상

### 시나리오 2: 힐링이 필요할 때

> "힘든 하루, 친구가 추천해주는 위로의 음악"

1. A가 "내가 힘들 때 들으면 좋을 음악은?" 선택
2. B가 A를 떠올리며 음악 선택
3. A가 받은 카드와 함께 음악으로 위로받음

### 시나리오 3: 가벼운 추천

> "친구가 요즘 꽂혀있는 음악이 궁금해!"

1. A가 "요즘 네가 꽂힌 음악은?" 선택
2. B가 최근 자주 듣는 곡 공유
3. A가 새로운 음악 취향 발견

## 경쟁 차별화 요소

### vs 스트리밍 서비스 추천

| 구분      | 스트리밍 앱   | MUKA                 |
| --------- | ------------- | -------------------- |
| 추천 방식 | 알고리즘 기반 | 친구의 직접 선택     |
| 감성 요소 | 없음          | 카드 디자인으로 표현 |
| 소셜 요소 | 제한적        | 1:1 특별한 경험      |

### vs 일반 링크 공유

| 구분      | 일반 공유 | MUKA                   |
| --------- | --------- | ---------------------- |
| 공유 형태 | 단순 링크 | 디자인된 카드          |
| 맥락      | 없음      | 질문으로 맥락 제공     |
| 특별함    | 없음      | 일회성으로 특별함 부여 |

### 핵심 차별점

1. **질문 기반 추천**: 맥락 있는 음악 공유
2. **감성 카드**: 단순 링크가 아닌 예쁜 컨텐츠
3. **일회성**: 특별하고 소중한 경험
4. **로그인 불필요**: 최소한의 진입 장벽
5. **카카오톡 최적화**: 한국 시장 친화적
